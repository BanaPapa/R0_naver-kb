## 배경 (30초 요약)

이 앱의 KB 시계열 모듈은 KB 데이터허브 엑셀 2개(주간/월간)를 JSON으로 변환해
Supabase Storage로 배포하고, 클라이언트가 버전 비교로 받아 차트·AI 분석에 쓴다.
문제: **① 엑셀→JSON 변환 스크립트가 저장소에 없어 매주 갱신이 재현 불가**,
② 인제스트된 데이터에 지역 매핑 결함 존재, ③ 월간 파일 54개 데이터 시트 중 12개 시트분만 활용,
④ 2026 행정구역 개편(인천 구 개편·전남광주통합) 컬럼이 곧 채워지는데 수용 불가.

원본 엑셀 위치(사용자 다운로드 폴더, 매주/매월 새 파일):
- `C:\Users\Space\Downloads\20260629_주간시계열.xlsx` (파일명 규칙: `YYYYMMDD_주간시계열.xlsx`)
- `C:\Users\Space\Downloads\202606_월간 주택 시계열.xlsx` (규칙: `YYYYMM_월간 주택 시계열.xlsx`)

현재 산출물: `public/data/kb-{weekly,weekly-trade,monthly,monthly-trade,monthly-forecast}.json`
발행: `scripts/kb-publish-bundles.mjs` (gzip → Supabase Storage `kb-data` 버킷 + versions.json)
클라이언트 로더: `src/kb/shared/lib/kb-source/` (변경 불필요 — 산출물 형태만 유지하면 됨)

## P0 — 재현 가능한 인제스트 파이프라인 (최우선)

`scripts/kb-ingest.mjs` 신규 작성. 목표: **엑셀 2개 경로를 인자로 받아 public/data/*.json 5개를
결정적으로 재생성**하고, 기존 산출물과의 diff 리포트를 출력한다.

```
node scripts/kb-ingest.mjs --weekly "…\20260629_주간시계열.xlsx" --monthly "…\202606_월간 주택 시계열.xlsx"
node scripts/kb-publish-bundles.mjs   # 기존 스크립트로 발행
```

### 구현 규칙 (보고서 §2·§3 근거)

1. **시트 스키마 테이블 기반**으로 작성(하드코딩 분기 금지). 시트별 정의:
   `{ 시트명, 헤더행, 데이터시작행, 날짜축 종류, 열 그룹 구조(플랫|지역당 N열|유형서브컬럼), 추출 열 }`
2. **날짜 파서 3종**: 엑셀 시리얼(주간) / 연도 전파형 `'86.1`→`1986-01` (월간, 연도는 바뀔 때만
   표기됨. `2016. 1` 공백 변형 포함) / 분기형 `'08 1Q` (PIR 계열 — P2에서만 필요).
3. **지역 정규화 테이블** `scripts/kb-regions.mjs` 분리:
   - 열 순서 기반 부모 추적(광역 라벨 뒤 구·시는 직전 광역의 하위). 동명 구(중구·서구 등)는
     반드시 `부모|이름` 키로.
   - canonical key 매핑: `전라북도→전북특별자치도`, `경북→경상북도`, `의왕→의왕시`,
     `제주도→제주특별자치도`(빈 컬럼은 실데이터 컬럼과 병합), `강원→강원특별자치도`.
   - 신설 지역 등록: 인천 `제물포구|영종구|서해구|검단구`, `전남광주통합특별시`(현재 전부 null —
     값이 생기면 자동 포함되도록 컬럼은 항상 파싱), 부천 3구·화성 4구는 `경기도|부천시|원미구`
     식으로 시 계층 유지.
4. **기존 결함 수정 확인**: 재생성된 `kb-weekly.json`에 `경기도|강원특별자치도도`,
   `경기도|춘천시` 류 유령 키가 없어야 한다(검증 스크립트에 assert 포함).
5. **검증 리포트**: 지역 수·기간·유효값 개수를 기존 JSON과 비교 출력. 값 차이가 0.1% 이상인
   시리즈는 나열(원천 정정 감지용).
6. 산출 JSON 스키마는 **기존과 동일하게 유지**(클라이언트 무수정). 단 `regions` 트리
   (kb-monthly.json에만 있음)를 주간에도 추가하는 것은 허용 — 로더가 무시하므로 안전.

### 갱신 워크플로 문서화

`docs/KB_UPDATE_RUNBOOK.md` 작성: ① kbland 통계 페이지에서 주간/월간 엑셀 다운로드(링크 명시:
https://kbland.kr/webview.html#/main/statistics?channel=kbland&tab=0 ) → ② kb-ingest 실행 →
③ diff 확인 → ④ kb-publish-bundles 실행. **자동화 여지**: 다운로드 자동화는 KB 로그인/약관 문제가
있으니 이번 범위에선 수동 유지, 대신 앱에 "데이터 기준일 + 다음 갱신 예정일" 표시(P2-3)로 보완.

## P2 — 분석 기능 강화 (상관분석 결과 반영, 보고서 §4 근거)

1. **AI 분석 프롬프트에 도메인 지식 주입** (`src/kb/features/analysis/prompts/analysis-prompt.md`):
   - 전망지수는 다음 달 매매증감과 최대 상관(전국 r≈0.82) → "향후 전망" 섹션 근거로 사용 지시.
   - 매매-전세 동행성: 지방 광역시 r≈0.9, 서울 핵심지 r≈0.55 → 지역 성격 구분 서술 지시.
   - 지역 전이: 서울→경기 +2주/인천 +4주, 서울↔지방 광역시는 비동행(r 0.2~0.3) —
     블록(수도권/지방) 밖 비교는 "비동행 자체가 정보"로 다루게 지시.
   - 매수우위·전세수급지수는 동행 지표(선행 아님)로만 인용하게 지시.
2. **비교군 프리셋**: 지역 선택 UI에 "수도권 블록(서울·경기·인천)", "지방 광역시 블록
   (부산·대구·대전·울산·광주)", "동행성 낮은 쌍(서울 vs 부산)" 원클릭 프리셋.
3. **데이터 신선도 표시**: KB탭 헤더에 각 데이터셋 기준일(주간 2026-06-29 / 월간 2026-05 형식)과
   "다음 갱신 예정(주간: 매주 금요일, 월간: 매월 말)" 뱃지. 기준일이 예정 주기보다 오래되면
   경고색 — 수동 갱신 파이프라인의 잊음 방지.

## 주의사항

- 클라이언트 로더/캐시(`kb-source/`)는 산출물 스키마를 유지하는 한 수정 불필요.
- Supabase service_role 키는 스크립트 환경변수로만(절대 커밋 금지) — 기존 publish 스크립트 관례 유지.
- 월간 시트 중 `★기간비교`·`★지역비교`·`안내`·`조사개요`·마지막 무명 시트(` `)는 파생/참조용이므로
  인제스트 대상에서 제외.
- 시트 31/35(구 면적별)는 2024-04에 중단된 시리즈 — 신기준 39/40을 사용.
